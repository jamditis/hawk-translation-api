Hawk is the translation engine built by the Center for Cooperative Media for NJ News Commons partner newsrooms. You submit an article. Hawk translates it, checks its quality, and delivers the result ‚Äî with your HTML formatting intact. Human editors can review flagged passages before anything goes live. Think of it as a wire service, but for translations.

No technical background needed to understand how it works. This page walks you through what happens from the moment a newsroom submits a story to the moment translated text lands in their CMS.

01
How it works

From article to translation in seven steps

When a newsroom partner sends a story to Hawk, it moves through a pipeline ‚Äî a set of automated stages that each do one specific job. Here's what happens at each stage.

1
Authentication

The newsroom's API key is checked

Every partner newsroom has a unique API key ‚Äî like a password that identifies them. Hawk checks the key instantly to confirm: is this a real partner? Do they still have translation credits available today? If yes, the job moves forward. If not, they get an immediate error message so they know what to fix.

2
Job creation

A translation job is created and queued

Hawk records the job in its database ‚Äî the original article text, the target language, the newsroom's preferences ‚Äî and immediately replies to the newsroom with a job ID. The newsroom doesn't have to wait. They get that ID right away and can check back for results. The actual translation happens in the background.

3
Segmentation

The article is broken into translatable pieces

News articles aren't plain text ‚Äî they contain HTML formatting (bold text, links, image captions, pull quotes). Hawk's segmenter carefully separates the readable text from the formatting code so only the words get translated, while all the layout instructions pass through untouched. After translation, the pieces are reassembled exactly as they were.

4
Glossary

NJ-specific terms are protected before translation

This is where local knowledge matters most. Hawk applies a custom journalism glossary before anything is sent to the translation engine. Proper nouns that shouldn't be translated ‚Äî like "Board of Education," "Paterson," or "NJ Transit" ‚Äî get locked in so they come through correctly in every language. STNS colleagues maintain this glossary.

5
Translation

DeepL translates the text

Hawk uses DeepL ‚Äî one of the most accurate machine translation services available ‚Äî for seven of the ten languages. DeepL was chosen because it handles journalistic prose better than most alternatives, producing more natural-sounding sentences. For Haitian Creole, Hindi, and Urdu (which DeepL doesn't support as well), segments are flagged for human review.

6
Quality scoring

Claude AI checks translation quality, paragraph by paragraph

After translation, Hawk runs each paragraph through a quality check using Claude ‚Äî an AI model made by Anthropic. Claude reads both the original English and the translated version and scores each paragraph on two things: fluency (does it read naturally in the target language?) and accuracy (is the meaning preserved?). Anything that scores below a 3 out of 5 gets flagged for human review. This happens quietly in the background and doesn't slow down the final delivery.

7
Delivery

The translated article is delivered

Once complete, the translated article is available for the newsroom to retrieve. If the newsroom provided a webhook URL (a web address where they want to receive the result automatically), Hawk sends it there the moment translation finishes. The newsroom can then publish it directly to their CMS. No manual copy-paste required.

02
The NJ journalism glossary

Why local terms matter ‚Äî and how Hawk protects them

Machine translation fails on local names. It may try to translate "Board of Education" into Spanish literally, or anglicize "Paterson" into something unrecognizable, or confuse government titles that have official translations. That's why Hawk applies a custom glossary before any text reaches the translation engine.

How glossary protection works

STNS maintains a list of terms that should always appear a specific way in each target language. Before translation, Hawk scans every paragraph and substitutes the correct version of each term. Longer phrases take priority ‚Äî "Board of Education" is matched before "Board" ‚Äî so no phrase gets partially overwritten.

05
Who uses Hawk

Built for two groups working together

NJ News Commons partner newsrooms

The 330+ newsrooms in the NJ News Commons network. They submit articles via the API (or, eventually, a WordPress plugin that handles it automatically). They get translations delivered directly back to their CMS. They don't need to manage the translation process ‚Äî just submit and receive.

STNS translation editors

The Spanish Translation News Service team at CCM. They maintain the glossary, handle reviewed and certified tier jobs, and approve final translations before delivery. Hawk surfaces only the passages that actually need human attention ‚Äî not the whole article.

06
What Hawk preserves

Your article's formatting comes out intact

One of the most common problems with translation tools is that they destroy formatting. Paste an article with bold text, hyperlinks, and image captions into most translators and you get back a mess of stripped tags and broken links.

Hawk handles this differently. It separates the readable words from the formatting code before translation, and reassembles the article afterward. The result: the same HTML structure goes in, and a fully translated version with identical formatting comes out. Bold stays bold. Links stay linked. Captions stay where they belong.

07
Getting started

How newsrooms get access

Access to Hawk is managed through the Center for Cooperative Media. Partner newsrooms in the NJ News Commons network can request an API key through CCM. STNS translation editors are onboarded directly.

For newsroom partners: Contact the Center for Cooperative Media to request API access. You'll receive an API key and documentation for connecting your CMS. A WordPress plugin that handles submissions automatically is in development.

For STNS translation editors: The review interface is browser-based ‚Äî no installation required. You'll be assigned jobs based on your language pair. Contact CCM to be added to the reviewer team.
